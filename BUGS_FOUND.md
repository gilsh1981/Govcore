# Bugs Found & Fixed — Phase 0

| Date | Bug | Files Changed |
|------|-----|---------------|
| 2026-05-29 | Create Committee dialog sent `chairId` but the API validator expects `chairUserId` — chairperson selection was silently discarded and committees were always created without a chair. | `src/components/committees/create-committee-dialog.tsx` |
| 2026-05-29 | Create Decision dialog sent field named `body` but the API validator expects `description` — decision descriptions were silently stripped by Zod and never saved. Also removed a phantom `dueDate` field that the Decision model doesn't have. | `src/components/decisions/create-decision-dialog.tsx` |
| 2026-05-29 | Create Meeting and Create Task dialogs swallowed API errors silently — `console.error` only, no user feedback. Added error state and inline error display to both dialogs. | `src/components/meetings/create-meeting-dialog.tsx`, `src/components/tasks/create-task-dialog.tsx` |
| 2026-05-29 | Upload API accepted file metadata but never wrote bytes to disk — files were silently lost. Added `src/lib/storage.ts` for local-disk I/O, a `PUT /api/upload/[sessionId]/files/[fileId]/content` route that receives `multipart/form-data` and writes to `uploads/{orgId}/{sessionId}/{fileId}`, and a `GET /api/documents/[id]/download` route to serve files back. | `src/lib/storage.ts`, `src/app/api/upload/[sessionId]/files/[fileId]/content/route.ts`, `src/app/api/documents/[id]/download/route.ts`, `.gitignore`, `.env` |
| 2026-05-29 | OrgSwitcher showed hardcoded mock org names ("Demo Cooperative") instead of the real organisation from the database. Fixed by storing `orgName` in the JWT at login, surfacing it in the session, and rewriting OrgSwitcher to use it. | `src/lib/auth.ts`, `src/types/index.ts`, `src/components/layout/org-switcher.tsx`, `src/components/layout/header.tsx` |
| 2026-05-29 | `<SelectItem value="">` crashes Radix Select — "value prop must not be empty string". Chairperson select in Create Committee and Assignee select in Create Task both used `value=""` as a sentinel for "no selection". Fixed by replacing with `value="none"` and updating submit logic to map `"none"` → `undefined`. | `src/components/committees/create-committee-dialog.tsx`, `src/components/tasks/create-task-dialog.tsx` |
